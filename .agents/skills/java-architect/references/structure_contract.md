# Structure Contract

## Document Section Hierarchy

The document follows a professional technical book structure.

### Front Matter (must be present)
1. **Cover Page** - Title, subtitle, author, graphic element
2. **Title Page & Copyright Page** - (optional, minimal)
3. **Table of Contents** - Hierarchical (H1/H2/H3) with dot leaders and page numbers
4. **Preface** - Explains purpose and what the reader will learn
5. **Who This Book Is For** - Target audience description

### Main Body (Parts and Chapters)
Organize content into **Parts** containing **Chapters**. Use Section Divider pages between Parts.

### Back Matter (must be present)
1. **References / Bibliography**
2. **Index** - (optional)

## Pattern Structure (Fowler-style)

Each pattern must follow this structure:
- **Intent**: Introductory paragraph explaining the pattern's purpose.
- **How It Works**: Detailed explanation of mechanics, components, and interactions.
- **When to Use It**: Scenarios where appropriate (complexity, team size).
- **Example**: Code example (properly indented, monospace, syntax-appropriate).
- **Consequences / Trade-offs**: Table of Positive vs Negative aspects.
- **Related Patterns**: Links to related patterns.

## Chapter Summary Structure (Hombergs-style)

At the end of each chapter, include:
### How Does This Help Me Build Maintainable Software?
A summary (2-4 paragraphs) recapping the architectural principle, practical benefits, real-world connection, and a bridge to the next topic.

## Code Block & Diagram Requirements
- **Language**: Java (primary) or C#.
- **Formatting**: 4 spaces indentation, each statement on its own line, monospace, light gray/blue background.
- **Captions**:
  - Code: *Listing N: Description* (below)
  - Figures: *Figure N: Description* (below, centered)
  - Tables: *Table N: Description* (above, left-aligned)
